QA Test Plan: Money Transfers API

1. User Creation API (POST /api/v1/users)

Title Steps Expected Result Status
TC-U01: Create valid user 1. Send POST with valid `name` and `email`. 201 Created. User is returned with a valid UUID. Not Executed
TC-U02: Missing required fields 1. Send POST with empty body {}. 422 Unprocessable Entity. Validation errors for `name` and `email`. Not Executed
TC-U03: Duplicate email 1. Create a user with `test@example.com`.
2. Send POST again with the exact same email.		 422 Unprocessable Entity. Validation error indicating email is already taken. Not Executed
TC-U04: Invalid email format 1. Send POST with email `invalid-email-format`. 422 Unprocessable Entity. Validation error for email format. Not Executed
TC-U05: Fake DNS/Disposable email (RFC, DNS Check) 1. Send POST with email `test@fake-domain-123456.xyz`. 422 Unprocessable Entity. Validation error indicating the email domain lacks a valid MX/A record. Not Executed

2. Account Creation API (POST /api/v1/accounts)

Title Steps Expected Result Status
TC-A01: Create valid account 1. Send POST with valid `user_id` (UUID), `name`, and `balance` (e.g., 100.50). 201 Created. Account is returned, balance matches exactly 100.50. Not Executed
TC-A02: Non-existent user ID 1. Send POST with a properly formatted UUID that does not exist in the users table. 422 Unprocessable Entity. Validation error indicating `user_id` does not exist. Not Executed
TC-A03: Negative initial balance 1. Send POST with `balance` = -50.00. 422 Unprocessable Entity. Validation error indicating balance must be >= 0. Not Executed

3. Money Transfer API (POST /api/v1/transfers)

Title Steps Expected Result Status
TC-T01: Valid transfer 1. POST transfer of $50.00 from Account A ($100) to Account B ($0). 200 OK. Transfer is `success`. Account A balance is now $50. Account B balance is $50. Not Executed
TC-T02: Exact balance transfer (Boundary) 1. POST transfer of exactly $100.00 from Account A ($100) to Account B ($0). 200 OK. Account A balance becomes $0.00. Account B balance becomes $100.00. Not Executed
TC-T03: Insufficient balance 1. POST transfer of $101.00 from Account A ($100) to Account B ($0). 422 Unprocessable Entity. Message: "Insufficient balance". Balances remain unchanged. Not Executed
TC-T04: Transfer to same account 1. POST transfer where `source_account_id` equals `destination_account_id`. 422 Unprocessable Entity. Validation error indicating source and destination must be different. Not Executed
TC-T05: Concurrency / Deadlock check (Edge Case) 1. Fire 10 simultaneous API requests sending $10 from Account A to Account B.
2. Fire 10 simultaneous API requests sending $10 from Account B to Account A.		 All requests resolve without 500 Deadlock errors. Final balances accurately reflect the net transfers. Not Executed

4. Transaction Ledger API (GET /api/v1/accounts/{id}/transactions)

Title Steps Expected Result Status
TC-L01: Fetch transaction history 1. Execute a valid transfer.
2. GET transactions for the Source Account.		 200 OK. Returns a `debit` transaction logging the exact amount, and `balance_before` / `balance_after`. Not Executed
TC-L02: Invalid account UUID 1. GET transactions for an account ID that does not exist. 404 Not Found. Proper JSON error message returned, not a stack trace. Not Executed

5. Security & Rate Limiting

Title Steps Expected Result Status
TC-S01: Rate Limit Trigger 1. Fire 61 API requests to any endpoint within a 60-second window from the same IP. Request 61 returns 429 Too Many Requests. Headers include `X-RateLimit-Reset`. Not Executed
Assumptions:
  • The API consumes and returns application/json exclusively.
  • Authentication is out-of-scope for this specific iteration (endpoints are publicly accessible for the test).
  • Database transactions and locks (lockForUpdate) are supported by the underlying active database connection (e.g., InnoDB on MySQL).
